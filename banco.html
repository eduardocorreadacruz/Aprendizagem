Conceitos de Banco de Dados

O que é um Banco de Dados?

Um banco de dados é uma coleção organizada de dados que pode ser facilmente acessada, gerenciada e atualizada. Ele é projetado para armazenar informações de maneira estruturada, permitindo o gerenciamento eficiente e a busca dos dados.

Banco de Dados Relacional

Um banco de dados relacional organiza os dados em tabelas, compostas por linhas e colunas, onde cada tabela representa uma entidade e as relações são definidas através de chaves.

  • Estrutura: Tabelas com linhas e colunas.
  • Exemplo: MySQL, PostgreSQL.
  • Manipulação: Utiliza SQL (Structured Query Language).

Banco de Dados Não Relacional

Os bancos de dados não relacionais, ou NoSQL, armazenam dados em formatos distintos como chave-valor, documentos, colunas ou grafos.

  • Exemplos: MongoDB (documento), Redis (chave-valor), Cassandra (coluna).
  • Flexibilidade: Estruturas de dados variadas.
  • Escalabilidade: Capazes de lidar com grandes volumes de dados.

O que é um Diagrama Conceitual (DER)?

Um Diagrama de Entidade-Relacionamento (DER) representa visualmente as entidades em um sistema e suas relações, auxiliando no design de banco de dados.

  • Entidades: Representam objetos do mundo real (ex.: Cliente, Produto).
  • Atributos: Características das entidades (ex.: Nome, Preço).
  • Relacionamentos: Como as entidades interagem (ex.: um Cliente faz um Pedido).

O que é um Diagrama Lógico (MER)?

Um Diagrama Lógico de Entidade-Relacionamento (MER) é uma versão detalhada do DER, mostrando chaves primárias, chaves estrangeiras e restrições de integridade.

O que é Dicionário de Dados?

Um dicionário de dados é um repositório que contém informações sobre os elementos de dados em um sistema, incluindo nomes, tipos, descrições e relacionamentos.

  • Nome do Campo: Nome do atributo ou coluna.
  • Tipo de Dados: Tipo de dado armazenado (ex.: inteiro, texto).
  • Descrições: Explicação sobre o significado do campo.
  • Restrições: Regras e limitações para os dados.
